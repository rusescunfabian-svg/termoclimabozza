================================================================================
  GITHUB PAGES — PAȘI (site în ROOT, cel mai simplu)
================================================================================

Pe PC, site-ul e în:  C:\Users\utente\termoclima-ab-site
Fișiere de urcat:    index.html, styles.css, script.js, privacy.html, favicon.svg, .nojekyll


1) REPO CURAT
   Deschide repository-ul pe GitHub. Dacă ai foldere ciudate („termoclima-ab-site”
   în interior) sau vechi fișiere: șterge tot (sau fă repo NOU).


2) ÎNCARCĂ ÎN ROOT
   Add file → Upload files
   Selectează din PC ACESTE 6 fișiere din folderul termoclima-ab-site și urcă-le.
   
   REZULTAT OBLIGATORIU: pe prima pagină a repo-ului vezi direct „index.html”
   (NU într-un subfolder gen termoclima-ab-site/index.html).


3) SETTINGS → PAGES
   • Source: „Deploy from a branch” ( dacă scrie „GitHub Actions”, SCHIMBĂ )
   • Branch: main  (sau master — la fel ca la repo )
   • Folder: / (root)   ← trebuie „root”, NU /docs
   • Save


4) URL (obligatoriu cu numele repo-ului)
   https://USER.github.io/NUME-REPO/

   Test direct:
   https://USER.github.io/NUME-REPO/index.html


5) Așteaptă 1–3 minute. Dacă e repo PRIVAT pe cont gratuit, Pages poate să nu
   meargă — fă repo PUBLIC la Settings → General → Danger zone (sau „Change visibility”).


================================================================================
GREȘELI care dau mereu 404:
• Deschizi doar https://USER.github.io/  fără /NUME-REPO
• Ai pages la /docs dar fișierele sunt în root (sau invers)
• index.html e în subfolder (ex. termoclima-ab-site/index.html) nu în root
