  TERMOCILMA A.B. — GitHub Pages FĂRĂ 404 (metoda recomandată)
================================================================================

METODA A — GITHUB ACTIONS (rezolvă și cazul „am încărcat în subfolder”)
------------------------------------------------------------------------

1. Pe PC, încarcă pe GitHub TOT conținutul folderului termoclima-ab-site, inclusiv:
   • .github/workflows/pages.yml   (OBLIGATORIU pentru această metodă)
   • index.html, styles.css, script.js, privacy.html, favicon.svg
   • .nojekyll (gol)
   • opțional: CARICA-GITHUB.txt

   Pe web: „Add file” → „Upload files” → poți trage tot folderul .github (verifică că
   în repo apare calea .github/workflows/pages.yml).

2. Su GitHub, nel repository: Settings → Pages.

3. La „Build and deployment” → Source: NU „Deploy from a branch”.
   Alege **GitHub Actions** (buton sau opțiune „GitHub Actions”).

4. Tab **Actions**: așteaptă workflow-ul „Deploy Pages” să fie verde (primele date
   poate trebui să accepți permisiunile: Settings → Actions → General →
   „Workflow permissions” → permiti read/write dacă îți cere).

5. URL site (project repo):
   https://TU-USER.github.io/NUME-REPO/
   Exemplu: https://ion.github.io/termoclima-ab-site/

6. NU deschide doar https://TU-USER.github.io/ — fără numele repository-ului e alt site
   sau 404.


METODA B — Deploy from branch (fără Actions)
--------------------------------------------------------------------------------
• index.html trebuie să fie în ROOT-ul repo-ului, NU într-un subfolder.
• Settings → Pages → Branch main → / (root).
• Dacă vezi în repo: termoclima-ab-site/index.html → e GREȘIT pentru metoda B.
  Folosește metoda A sau mută fișierele în root.
